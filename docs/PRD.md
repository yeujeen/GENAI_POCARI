## 1. 프로젝트 개요

- ‘Friendship FUSE (프렌드십 퓨즈)’는 AI가 두 사용자의 음악 취향 데이터를 분석하여 공통 관심사를 시각화하고, 최적의 공동 플레이리스트를 자동 생성하여 음악을 통한 사회적 연결을 강화하는 소셜 음악 서비스입니다.

---

## 2. 문제 정의 및 솔루션

### 2.1 핵심 문제

- **높은 음악 취향 공유 허들** : 친구와 취향이 다를까 봐 플레이리스트 공유를 망설이거나, 공유해도 서로에게 낯선 곡이 너무 많아 흥미를 잃음.
- **'교집합'의 부재 :** 두 사람의 취향 데이터를 단순 합산만 할 뿐, '우리가 함께 들을 수 있는' 공통 영역을 찾고 시각적으로 보여주는 서비스 부족
- **비효율적인 믹싱 :** 함께 들을 리스트를 만들 때 수동으로 곡을 선별해야 하는 시간 및 노력 낭비 발생

### 2.2 솔루션 가설

“AI 기반으로 두 사용자 데이터의 **교집합(공통 태그/장르/아티스트)을 기반**으로 **공동 플레이리스트(FUSE MIX)**를  자동 생성하고 취향 유사도를 직관적으로 시각화하면, 음악을 통한 **관계 강화와 공동 청취 만족도**가 향상될 것이다."

---

## 3. 핵심 기능

### (1) 데이터 연결 및 분석

두 사용자의 가상 청취 데이터를 입력/연동하고, **장르, 아티스트, 감성 태그 기반**의 **취향 유사도(%)**를 산출.

### (2) 취향 분석 및 매칭

장르, 아티스트, BPM 태그 기반 **취향 유사도(%)** 측정 및 공통 곡 추천 리스트 도출 

### (3) FUSE Mix 생성 및 시각화

교집합 영역의 10~15곡 자동 플레이리스트 생성, **벤 다이어그램**으로 취향 교집합 시각화**,** 결과 공유 기능(**고유 URL** 제공)

---

## 4. 사용자 시나리오

**팀플을 준비 중인 대학생 User A와 User B**

1. User A가 User B에게 'Friendship FUSE' 링크 공유
2. User B가 자신의 음악 데이터를 연동
3. AI가 "유사도 45%" 계산 및 벤 다이어그램으로 시각화 (자카드 유사도)
4. '팀플 집중 믹스'라는 공동 플레이리스트 자동 생성
5. 두 사람이 함께 플레이리스트를 청취하며 작업, 결과를 SNS에 공유

---

## 5. 기대 효과

### 사용자 측면

- 음악 취향 공유 과정의 재미와 효용성 증대로 서비스 재이용률 상승
- 취향 차이 장벽을 넘어 음악을 통한 **관계 강화** 경험
- 공동 청취를 위한 **시간 및 노력 절약**

### 사회적 측면

- 음악 소비를 **개인화에서 소셜** 경험으로 확장하는 새로운 트렌드 제시

### 비즈니스 측면

- 직관적이고 재미있는 시각화 결과를 통한 바이럴 마케팅 효과 및 사용자 충성도 확보
- 음악 스트리밍 플랫폼과의 파트너십 가능성
- 프리미엄 구독 모델(고급 분석 지표 제공 등) 수익 창출 기능.

## 7. 기술 스택

### 1. 핵심 AI 모델 (두뇌)

- **Google Gemini API**
    - **역할 (All-in-One):** 이것이 사실상 MVP의 전부입니다.
    - **기능 1 (분석):** 두 사용자의 취향 데이터(예: 좋아하는 아티스트 5명씩)를 프롬프트로 전달받아 **유사도(%)를 추정**합니다.
    - **기능 2 (생성):** 분석 결과를 바탕으로 **'FUSE Mix' (공동 플레이리스트 10곡)를 생성**합니다.
    - **기능 3 (스토리텔링):** 생성된 믹스에 대한 **재미있는 제목과 설명 문구**를 생성합니다.

### 2. 개발 및 실행 환경 (UI 및 서버)

- **Next.js (Bolt.new 자동 생성)**
    - **역할:** 사용자가 취향을 입력할 **웹페이지(프론트엔드)**와 위 AI 모델을 호출할 **통신 창구(백엔드 API)** 역할을 동시에 수행합니다.
- **v0.dev**
    - **역할:** UI 디자인 및 코드 생성을 도와 개발 시간을 단축시킵니다.

### 3. 배포

- **Vercel** (Next.js와 호환성이 가장 좋음) 또는 **Replit**
    - **역할:** 완성된 웹 서비스를 실제 인터넷 주소(URL)로 배포하여 다른 사람들이 사용할 수 있게 합니다.
